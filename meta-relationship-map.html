Meta 广告数据接入关系图

核心结论：不是 Facebook 个人号直接读取广告数据，而是业务资产组合里的系统用户，拿到某个 App 生成的带 ads_read 权限的 Token，再去读取广告账户 act_xxx 的 Insights。

目标：Ads Insights 最小权限：ads_read 账户 ID：act_xxx
API

你要打通的是这一条链路

Facebook 个人号只负责登录和管理；真正给程序用的是 System User Token。这个 Token 必须来自一个 App，并且这个系统用户必须被分配到目标广告账户。

能进后台 说明个人号或业务身份有界面权限，不等于 API 已经能读。
能生成 Token 说明 App、系统用户、资产权限已经连上，但还要看有没有 ads_read。
能拉报表 需要请求 /act_广告账户ID/insights 成功返回 spend、clicks、actions、ROAS 等字段。

一句话版

FB 号 是人，BM/业务资产组合 是公司资产柜，广告账户 是花钱投放的账户，广告系列 是账户里的投放结构，App 是 API 身份，系统用户 是后端机器人，角色/权限 决定谁能碰哪些资产。

  • FB
    Facebook 个人号
    真实自然人账号，用来登录、验证身份、加入业务资产组合。
    人
    • DEV
      Meta 开发者身份
      同一个个人号完成开发者注册后获得的开发者资格，不是另一个独立账号。
      资格
      • APP
        Meta Developer App
        API 调用的应用身份。读取广告报表通常要 Marketing API 和 ads_read。
        API 身份
    • BM
      Business Manager / Business Portfolio
      业务资产组合，集中管理广告账户、Pixel、Page、App、人员和系统用户。
      资产柜
      • ROLE
        人员、合作伙伴、角色
        管理员、员工、合作伙伴等角色，决定界面里能看什么、能改什么。
        权限
      • BOT
        系统用户 System User
        给服务器或脚本用的业务用户。它不登录网页，只拿 Token 调 API。
        机器人
        • TOK
          System User Token
          由某个 App 生成，权限里要包含 ads_read，才能读广告报表。
          钥匙
      • APP
        已加入 BM 的应用
        App 要在同一个业务资产组合里，系统用户才方便用它生成访问口令。
        应用资产
      • ACT
        广告账户 Ad Account
        真正产生花费、曝光、点击、转化报表的账户，API 里写作 act_123。
        投放账户
        • CMP
          广告系列 Campaign
          投放目标和顶层结构。你的截图第一列“广告系列”就是这一层。
          一级结构
          • SET
            广告组 Ad Set
            通常控制受众、版位、排期、预算或出价等。
            二级结构
            • AD
              广告 Ad
              具体素材、文案、落地页和创意。
              素材层
        • INS
          Insights 报表数据
          spend、clicks、ctr、actions、purchase_roas 等都从这里取。
          你要的数据
      • PIX
        Pixel / Dataset
        提供购买、加购、发起结账等转化事件。没有事件，ROAS 和电商转化会缺失。
        转化源

读取广告数据的最小路径

1
App 在 BM 里已有 App 可绑定；没有 App 才需要创建。
2
系统用户有广告账户权限至少要能查看广告账户表现数据。
3
生成含 ads_read 的 Tokenread_ads_dataset_quality 不等于 ads_read。
4
请求 Insights/act_广告账户ID/insights 返回报表字段。

到底要不要创建 App

只手动看报表

不用创建 App。直接在 Ads Manager 导出 CSV 或 Excel。

现有 App 能给 ads_read

不用新建。把现有 App、系统用户、广告账户权限连好即可。

现有 App 没有 ads_read

要么让管理员给你一个可用 App，要么完成开发者验证后新建 Business App。

常见误解

误解 正确理解
我能登录广告后台，所以 API 肯定能读。 不一定。后台权限和 API Token 权限是两回事。
新注册一个 FB 号会更干净。 不建议。Meta 对新号和多个人号更敏感，应该用真实稳定账号。
App 被拉进 BM 就能读广告数据。 还需要系统用户、广告账户权限、Token 权限同时满足。
有 CAPI 权限就能读广告报表。 不一定。CAPI/数据集质量权限和 Ads Insights 的 ads_read 不同。
